# 自動車保険見積サイト (Car Insurance Quote System)

本プロジェクトは、株式会社ティーアンドエス（T&S）の自動車保険見積サイト開発課題に基づき実装されたWebアプリケーションです。  
一般利用者が画面ガイダンスに沿って条件を入力し簡易保険料を算出する機能と、管理者が履歴の検索・CSV出力・料率マスタの参照を行う機能を備えています。

---

## 📌 技術スタック (Tech Stack)

- **Backend**: Java 21 / Spring Boot 3.3.0 / Spring Data JPA / Spring Security / Spring Cache (Simple) / Validation / H2 (テスト用)
- **Frontend**: React 18 / TypeScript / Vite / Vanilla CSS (カスタムグリッド・グラスモフィズム) / Lucide React (アイコン)
- **Database**: PostgreSQL 16 (本番・Docker) / H2 In-Memory (単体テスト)
- **API Spec**: Springdoc OpenAPI (Swagger UI) 2.5.0
- **Container**: Docker / Docker Compose

## 🚀 起動手順 (Execution Guide)

### 1. Docker Compose による一括起動（推奨・加点項目）
Dockerがインストールされている環境であれば、フロント・バック・DBを完全に自動連動させて一括起動できます。
```bash
# 1) 環境変数ファイルの準備（初回のみ）
cp .env.example .env
# 必要に応じて .env 内のDB接続情報やJWTシークレットを変更してください

# 2) dockerディレクトリへ移動
cd docker

# 3) コンテナの一括ビルドとバックグラウンド起動
docker compose up --build -d
```
- **フロントエンド（Nginx公開）**: `http://localhost`
- **バックエンド（API）**: `http://localhost:8080`
- **Swagger UI**: `http://localhost:8080/swagger-ui/index.html`
- **データベース**: `localhost:5432` に自動でPostgreSQLが起動し、初期データが投入されます。
- ※ `depends_on` に `healthcheck` 条件を指定しており、DBが完全に立ち上がり健康（Healthy）になってからバックエンドが起動するよう制御しています。
- ※ 完全初期化する場合は `docker compose down -v` で永続化ボリュームも削除してから再起動してください。

---

### 2. ローカル開発環境での手動起動

#### (1) データベース (PostgreSQL 16)
ローカルでPostgreSQL 16を起動し、以下のデータベースと接続情報を準備してください。
- **データベース名**: `quotesdb`
- **ユーザー名**: `quoteuser`
- **パスワード**: `quotepassword`
- ※ 起動時、バックエンドの `DatabaseInitializer` が自動的にテーブルの有無を検知し、データがなければマスタデータ (`rate_masters`) と管理者アカウント (`admin`/`admin123`) を自動挿入します。

#### (2) バックエンド (Spring Boot)
```bash
cd backend
# アプリケーションの起動 (デフォルトでdevプロファイルがアクティブになります)
mvn spring-boot:run
```
- 起動成功後、**Swagger UI** にアクセスしてAPI仕様を確認・テストできます。
- URL: `http://localhost:8080/swagger-ui/index.html`

#### (3) フロントエンド (React)
```bash
cd frontend
# 依存パッケージのインストール
npm install
# 開発サーバーの起動
npm run dev
```
- 起動後、ブラウザで `http://localhost:5173` にアクセスしてください。

## 🔐 管理者認証仕様

- **初期ユーザーID**: `admin`
- **初期パスワード**: `admin123`
- パスワードはデータベース上には平文で保存されず、**BCrypt強固ハッシュ**を用いて暗号化されています。
- 管理者画面 (`/api/admin/**`) のリクエストは、ログイン時に発行される **JWT (JSON Web Token)** をヘッダーに付与して認証します（`Authorization: Bearer <TOKEN>`）。

## 💡 設計上の工夫・品質向上ポイント (加点ポイント)

1. **マスタテーブル駆動型（Data-Driven）の計算設計**
   - 保険料率や特約額をJavaコード上に直接記述（ハードコーディング）せず、すべて `rate_masters` テーブルに定義しています。将来の料率変更時にも、**コードの変更・再ビルド不要**で、データベース設定値を更新するだけで即座に反映される保守性の高い設計となっています。
2. **スプリングキャッシュ（Spring Cache）の活用によるパフォーマンス改善**
   - 頻繁に参照され変更の少ない `rate_masters` のデータ取得メソッドに `@Cacheable` 缓存を適用。計算処理のたびに発生するDBアクセスを最小限に抑え、API全体のレスポンス速度を高速化しています。
3. **Nginx リバースプロキシによる CORS 回避と一元公開**
   - Docker環境下では、フロントエンド用のNginxコンテナが80ポートでリクエストを受け付け、`/api/` 配下のアクセスのみバックエンドへリバースプロキシする構成をとっています。これにより、ブラウザでの **CORS（オリジン間リソース共有）制約を綺麗に回避** しつつ安全に通信できます。
4. **Excel 文字化けを防ぐ UTF-8 BOM付き CSV出力**
   - 管理画面のCSV出力では、日本語版Microsoft Excelで読み込んだ際にお車の日本語名などが文字化けしないよう、ストリーム先頭に **BOM（Byte Order Mark: `0xEF, 0xBB, 0xBF`）** を明記して出力する細やかな配慮を行っています。
